Defining Member Functions Outside Class

This is the most important use.

Example

        #include <iostream>
        using namespace std;

        class Employee
        {
        private:
            int id;

        public:
            void setId(int i);
            void showId();
        };

        void Employee::setId(int i)
        {
            id = i;
        }

        void Employee::showId()
        {
            cout << "ID: " << id;
        }

        int main()
        {
            Employee e1;

            e1.setId(1001);
            e1.showId();

            return 0;
        }

Understanding
    Inside class

    Only declaration:
        void setId(int i);

    Outside class
        Definition using ::
        void Employee::setId(int i)

    Meaning:

        Function setId belongs to Employee class


---------------------------------------------------------------------------

Why Define Functions Outside Class?

Advantages:

    Cleaner code
    Better readability
    Easier maintenance
    Used in large projects
