Polymorphism is one of the most important OOP concepts because it shows the real power of object-oriented programming.

The easiest way to teach students is:

"One thing can behave in many forms."

The word itself means:

        Poly = Many
        Morph = Forms

        Polymorphism = Many Forms


    1. Introduction to Polymorphism

Simple Explanation

Polymorphism allows the same function or object to perform different actions in different situations.

    Real-World Example

Mobile Phone

One mobile phone can be used as:

        Phone
        Camera
        Calculator
        Music Player
        GPS

Same device → Multiple behaviors.

This is Polymorphism.

Types of Polymorphism


        Polymorphism
            |
            +---- Compile-Time Polymorphism
            |           |
            |           +---- Function Overloading
            |           +---- Operator Overloading
            |
            +---- Runtime Polymorphism
                        |
                        +---- Virtual Function
                        +---- Abstract Class
                        +---- Pure Virtual Function


COMPILE-TIME POLYMORPHISM

Compiler decides which function to execute before the program runs.

2. Function Overloading
Simple Explanation

Multiple functions can have:

        Same Name
        Different Parameters


Real-World Example


Calculator

The same "Add" button can add:

        2 Numbers
        3 Numbers
        Decimal Numbers

Same operation.

Different inputs.

Program


        #include <iostream>
        using namespace std;

        class Calculator
        {
        public:
            int add(int a, int b)
            {
                return a + b;
            }

            int add(int a, int b, int c)
            {
                return a + b + c;
            }
        };

        int main()
        {
            Calculator c;

            cout << c.add(10,20) << endl;

            cout << c.add(10,20,30);

            return 0;
        }


Output
        30
        60


Student Understanding


        add(10,20)
                ↓
        Function 1

        add(10,20,30)
                ↓
        Function 2

Compiler chooses automatically.

3. Operator Overloading


Simple Explanation

Operator Overloading means:

Giving a new meaning to an existing operator.

Real-World Example
'+' Operator

        Normally:

        10 + 20 = 30

        But:

        "Hello" + "World"

        means join two strings.

        Same operator.

        Different behavior.

Program


        #include <iostream>
        using namespace std;

        class Number
        {
        public:
            int value;

            Number(int v)
            {
                value = v;
            }

            Number operator +(Number obj)
            {
                return Number(value + obj.value);
            }
        };

        int main()
        {
            Number n1(10);
            Number n2(20);

            Number n3 = n1 + n2;

            cout << n3.value;

            return 0;
        }


Output
    
    30


Student Understanding

Normally:

10 + 20

Now:

Object + Object

is also possible.



RUNTIME POLYMORPHISM

Simple Explanation

Decision is taken while program is running.

Compiler doesn't know beforehand.

Real-World Example
Remote Control

The same remote can control:

        TV
        AC
        Projector

Depending on what device is connected.

Decision happens at runtime.

4. Virtual Function
Simple Explanation

A virtual function allows a child class function to replace the parent class function.

Real-World Example
Animal Sounds

All animals make sounds.

But:

        Dog → Bark
        Cat → Meow
        Cow → Moo

Same action.

Different implementation.

Program


        #include <iostream>
        using namespace std;

        class Animal
        {
        public:
            virtual void sound()
            {
                cout << "Animal Sound" << endl;
            }
        };

        class Dog : public Animal
        {
        public:
            void sound()
            {
                cout << "Dog Barks" << endl;
            }
        };

        int main()
        {
            Animal *ptr;

            Dog d;

            ptr = &d;

            ptr->sound();

            return 0;
        }


Output
Dog Barks
Student Understanding

Without virtual:

        Animal Sound

With virtual:

        Dog Barks

        The child version runs.



5. Abstract Class

Simple Explanation

An Abstract Class is an incomplete class.

It is only used as a blueprint.

Objects cannot be created directly.

Real-World Example
Building Blueprint

Blueprint tells:

        Rooms
        Doors
        Windows

But you cannot live inside a blueprint.

Actual building must be created.

Example
class Shape
{
public:
    virtual void draw() = 0;
};

This class is incomplete.

Student Understanding
Blueprint
     ↓
Actual Building

Similarly:

Abstract Class
      ↓
Derived Class
6. Pure Virtual Function
Simple Explanation

A pure virtual function forces child classes to implement it.

Syntax
virtual void show() = 0;

The =0 makes it pure virtual.

Real-World Example
Examination Rules

College says:

Every student must write exam

How they write:

Online
Offline
Computer Based

depends on the student.

But writing exam is mandatory.

Program
#include <iostream>
using namespace std;

class Shape
{
public:
    virtual void draw() = 0;
};

class Circle : public Shape
{
public:
    void draw()
    {
        cout << "Drawing Circle" << endl;
    }
};

int main()
{
    Circle c;

    c.draw();

    return 0;
}
Output
Drawing Circle
Student Understanding

Parent says:

Every shape must draw itself

Child decides:

Circle Draw
Rectangle Draw
Triangle Draw
Complete Example (Virtual + Abstract + Pure Virtual)

This is a very good interview example.

#include <iostream>
using namespace std;

class Vehicle
{
public:
    virtual void start() = 0;
};

class Car : public Vehicle
{
public:
    void start()
    {
        cout << "Car Started" << endl;
    }
};

class Bike : public Vehicle
{
public:
    void start()
    {
        cout << "Bike Started" << endl;
    }
};

int main()
{
    Vehicle *v;

    Car c;
    Bike b;

    v = &c;
    v->start();

    v = &b;
    v->start();

    return 0;
}
Output
Car Started
Bike Started
