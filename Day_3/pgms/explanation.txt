Constructors and Destructors in C++

(Simple Explanation Like Teaching a 5-Year-Old 😊)

First Understand This

    Imagine you bought a toy robot.

    When the robot is created:

        It gets a name
        It gets a color
        It says “Hello!”

This setup work happens automatically.

In C++, this automatic setup is done using a:

CONSTRUCTOR

    What is a Constructor?

    A constructor is a special function that runs automatically
     when an object is created.

Real-Life Example
    You buy a mobile phone
    ↓
    Phone starts automatically
    ↓
    Setup screen appears
    ↓
    Initialization happens

Same in C++:

    Object created
    ↓
    Constructor runs automatically
    ↓
    Object gets initialized


Important Rules of Constructor

    Rule	                            Meaning
    Constructor name = class name	    Same name
    No return type	                    Not even void
    Runs automatically	                No need to call manually

BASIC Constructor Example

    #include <iostream>
    using namespace std;

    class Car
    {
    public:
        Car()
        {
            cout << "Car Started!" << endl;
        }
    };

    int main()
    {
        Car c1;

        return 0;
    }

Output

Car Started!

Why Did It Print Automatically?

Because:

    Car()

        is a constructor.

It runs automatically when:

    Car c1;

        creates the object.

There are mainly 3 important types:

    Default Constructor
    Parameterized Constructor
    Copy Constructor

What is it?

A constructor with NO parameters.

Think Like a Child

2. PARAMETERIZED CONSTRUCTOR
What is it?

A constructor that accepts values.

Child Example

You order a toy car:

Color = Blue
Name = Racing Car

Factory builds exactly what you asked.

Example
#include <iostream>
using namespace std;

class Student
{
public:
    string name;

    Student(string n)
    {
        name = n;
    }

    void show()
    {
        cout << "Student Name: " << name << endl;
    }
};

int main()
{
    Student s1("Rahul");

    s1.show();

    return 0;
}
Output
Student Name: Rahul
Understanding

This value:

"Rahul"

goes into:

Student(string n)

Then:

name = n;

stores it.

Visual Flow
Student s1("Rahul")
↓
Constructor receives Rahul
↓
name = Rahul
3. COPY CONSTRUCTOR
What is it?

Copies one object into another object.

Child Example

Imagine:

One chocolate box exists
↓
Machine makes exact duplicate copy

That is copy constructor.

Example
#include <iostream>
using namespace std;

class Box
{
public:
    int value;

    Box(int v)
    {
        value = v;
    }

    // Copy Constructor
    Box(const Box &b)
    {
        value = b.value;
    }

    void show()
    {
        cout << "Value: " << value << endl;
    }
};

int main()
{
    Box b1(100);

    Box b2 = b1;

    b2.show();

    return 0;
}
Output
Value: 100
Understanding Slowly
Box b2 = b1;

means:

Make a copy of b1 into b2

Copy constructor does that work.

Important Syntax
ClassName(const ClassName &obj)
Why Use & ?

Because we do NOT want to copy again and again.

It saves memory.

DESTRUCTOR

Now objects are created.

But what happens when object dies?

Cleaning work is needed.

That cleaning work is done by:

Destructor
What is Destructor?

A special function that runs automatically when object is destroyed.

Child Example

Imagine playing with toys.

After playing:

Put toys back into box
Clean the room

Cleanup work = Destructor

Important Rules
Rule	Meaning
Same name as class	Yes
Uses ~ symbol	Yes
No return type	Yes
No parameters	Yes
Destructor Syntax
~ClassName()
{
}
Simple Destructor Example
#include <iostream>
using namespace std;

class Test
{
public:
    Test()
    {
        cout << "Constructor Called" << endl;
    }

    ~Test()
    {
        cout << "Destructor Called" << endl;
    }
};

int main()
{
    Test t1;

    return 0;
}
Output
Constructor Called
Destructor Called
Understanding

When object created:

Constructor runs

When program ends:

Object destroyed
↓
Destructor runs
REAL MEMORY MANAGEMENT IDEA
Why Destructor Important?

Programs use memory.

Unused memory should be cleaned.

Otherwise:

Memory gets wasted
System becomes slow

Destructor helps clean memory.

Child Example

Imagine:

You keep toys everywhere
Never clean
Room becomes full

Same for computer memory.

Destructor and Dynamic Memory
Example
#include <iostream>
using namespace std;

class Demo
{
private:
    int *ptr;

public:
    Demo()
    {
        ptr = new int;

        cout << "Memory Allocated" << endl;
    }

    ~Demo()
    {
        delete ptr;

        cout << "Memory Released" << endl;
    }
};

int main()
{
    Demo d1;

    return 0;
}
Output
Memory Allocated
Memory Released
Understanding Step-by-Step
Constructor
ptr = new int;

creates memory.

Destructor
delete ptr;

cleans memory.

MOST IMPORTANT INTERVIEW QUESTION
What happens if destructor is NOT used?

Answer:

Memory Leak happens
What is Memory Leak?

Unused memory remains occupied.

Program wastes RAM.

Final Combined Example
#include <iostream>
using namespace std;

class Toy
{
public:
    string name;

    // Default Constructor
    Toy()
    {
        name = "Default Toy";

        cout << "Default Constructor" << endl;
    }

    // Parameterized Constructor
    Toy(string n)
    {
        name = n;

        cout << "Parameterized Constructor" << endl;
    }

    // Copy Constructor
    Toy(const Toy &t)
    {
        name = t.name;

        cout << "Copy Constructor" << endl;
    }

    void show()
    {
        cout << "Toy Name: " << name << endl;
    }

    // Destructor
    ~Toy()
    {
        cout << "Destructor Called" << endl;
    }
};

int main()
{
    Toy t1;

    Toy t2("Car");

    Toy t3 = t2;

    t1.show();
    t2.show();
    t3.show();

    return 0;
}
Output
Default Constructor
Parameterized Constructor
Copy Constructor
Toy Name: Default Toy
Toy Name: Car
Toy Name: Car
Destructor Called
Destructor Called
Destructor Called
