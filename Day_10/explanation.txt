--------------------------------------------------------------------------------
Explanation for a 5-Year Old
--------------------------------------------------------------------------------
How is a human brain different from a computer brain? 
If you see a picture of a cute puppy, you instantly know it's a dog and say, "Aww!" But a computer only sees millions of colored dots. It has to look very closely at the dots to guess: "I think this is a dog!"

AI has different parts, like different jobs:
- **NLP (Language Talking)**: Helping the computer read stories and talk to us.
- **Computer Vision (Seeing)**: Giving the computer eyes to see cars, dogs, and faces.
- **Robotics (Moving)**: Giving the computer arms and legs to walk and carry things.

Sometimes computers make mistakes. If we only teach a robot using pictures of red cars, it might think blue cars are not cars at all! This is called **Bias** (an unfair mistake). We need to teach computers carefully so they are fair, respect our privacy, and help us do work without taking all our fun away!

--------------------------------------------------------------------------------
Technical Explanation
--------------------------------------------------------------------------------
Day 10 focuses on the categories of AI, practical integration, and its societal impact.

1. AI vs. Human Intelligence:
   - Human Intelligence: Broad, relies on common sense, emotional consciousness, and adapts to completely unseen environments seamlessly.
   - Artificial Intelligence: Task-focused, relies on mathematical patterns/data, and lacks emotional awareness or consciousness.

2. Major Branches of AI:
   - Natural Language Processing (NLP): Techniques to allow computers to understand, interpret, and generate human languages (e.g., translation, sentiment analysis).
   - Computer Vision: Training computers to extract information and understand visual data (images, videos) using algorithms (like Convolutional Neural Networks).
   - Robotics: Engineering field that builds physical robots that can navigate, perceive, and manipulate objects using sensory inputs.
   - Expert Systems: Computer systems that emulate the decision-making ability of a human expert using a database of facts and rules.

3. Automation vs. Intelligence:
   - Automation: Running a pre-programmed, fixed set of steps (e.g., a factory arm moving back and forth). No decision making is involved.
   - Intelligence: Responding dynamically to changing inputs and optimizing for a goal (e.g., a robot avoiding a obstacle in its path).

4. Ethical Concerns in AI:
   - Bias: Machine learning models reflect the biases present in their training datasets.
   - Privacy: AI requires massive datasets (often personal user data) to train effectively, raising privacy protection concerns.
   - Job Impact: Automation and AI agents replacing human labor in certain industries.

### Sample 1: Simple Natural Language Processing (NLP) Tokenizer
This C++ program shows how computers start processing text by splitting sentences into words (tokenization) and filtering out common meaningless words ("stop words" like "is", "a", "the").

```cpp
#include <iostream>
#include <string>
#include <vector>
#include <sstream>
#include <algorithm>

// Function to check if a word is a "stop word"
bool isStopWord(const std::string& word) {
    const std::vector<std::string> stopWords = {"is", "a", "the", "an", "on", "in", "to", "and"};
    return std::find(stopWords.begin(), stopWords.end(), word) != stopWords.end();
}

int main() {
    std::string text = "Artificial Intelligence is the future of computing and robotics";
    std::cout << "Original Text: " << text << std::endl;

    std::stringstream ss(text);
    std::string word;
    std::vector<std::string> tokens;

    // Tokenization: Splitting by spaces
    while (ss >> word) {
        // Convert to lowercase
        std::transform(word.begin(), word.end(), word.begin(), ::tolower);
        tokens.push_back(word);
    }

    std::cout << "\nTokens after Tokenization & Stop-word Filtering:" << std::endl;
    for (const std::string& t : tokens) {
        if (!isStopWord(t)) {
            std::cout << "[" << t << "] ";
        }
    }
    std::cout << std::endl;

    return 0;
}
```

### Sample 2: Robot Behavior State Machine (Decision Making & Robotics)
This C++ program simulates a robot's decision-making system using a Finite State Machine (FSM). The robot shifts between IDLE, EXPLORING, and CHARGING states based on its battery and sensor readings.

```cpp
#include <iostream>
#include <string>

enum State { IDLE, EXPLORING, CHARGING };

class Robot {
private:
    State currentState;
    int batteryLevel;

public:
    Robot() : currentState(IDLE), batteryLevel(100) {}

    void update(bool obstacleDetected) {
        std::cout << "\n--- Robot Status ---" << std::endl;
        std::cout << "Battery: " << batteryLevel << "%" << std::endl;

        switch (currentState) {
            case IDLE:
                std::cout << "Current State: IDLE" << std::endl;
                if (batteryLevel > 20) {
                    currentState = EXPLORING;
                    std::cout << "Action: Battery is good. Transitioning to EXPLORING." << std::endl;
                } else {
                    currentState = CHARGING;
                    std::cout << "Action: Battery is low! Transitioning to CHARGING." << std::endl;
                }
                break;

            case EXPLORING:
                std::cout << "Current State: EXPLORING" << std::endl;
                batteryLevel -= 15; // Exploring consumes battery

                if (batteryLevel <= 20) {
                    currentState = CHARGING;
                    std::cout << "Action: Battery critically low! Heading to charging dock." << std::endl;
                } else if (obstacleDetected) {
                    std::cout << "Action: Obstacle detected! Changing route direction." << std::endl;
                } else {
                    std::cout << "Action: Navigating room smoothly." << std::endl;
                }
                break;

            case CHARGING:
                std::cout << "Current State: CHARGING" << std::endl;
                batteryLevel += 40; // Charging restores battery

                if (batteryLevel >= 100) {
                    batteryLevel = 100;
                    currentState = IDLE;
                    std::cout << "Action: Fully charged! Transitioning to IDLE." << std::endl;
                } else {
                    std::cout << "Action: Still charging..." << std::endl;
                }
                break;
        }
    }
};

int main() {
    Robot myRobot;
    
    // Simulate robot lifetime steps
    myRobot.update(false); // IDLE -> EXPLORING
    myRobot.update(false); // EXPLORING (battery 85%)
    myRobot.update(true);  // EXPLORING (battery 70%, encounters obstacle)
    myRobot.update(false); // EXPLORING (battery 55%)
    myRobot.update(false); // EXPLORING (battery 40%)
    myRobot.update(false); // EXPLORING (battery 25%)
    myRobot.update(false); // EXPLORING -> CHARGING (battery 10%)
    myRobot.update(false); // CHARGING (battery 50%)
    myRobot.update(false); // CHARGING (battery 90%)
    myRobot.update(false); // CHARGING -> IDLE (battery 100%)
    
    return 0;
}
```
