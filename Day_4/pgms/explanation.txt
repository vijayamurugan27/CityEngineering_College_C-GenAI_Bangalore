1. Encapsulation (Data Hiding)
Simple Explanation

Encapsulation means keeping data and methods together inside a class and protecting the data from direct access.

Think of it as a capsule that hides the internal details.

In C++, we use private variables to hide data and public functions to access it.

Real-World Example
ATM Machine

When you withdraw money from an ATM:

You cannot directly access the bank database.
You enter your PIN.
The ATM checks everything internally.

The bank balance is hidden from you.

This is called Data Hiding (Encapsulation).

Simple Program
#include <iostream>
using namespace std;

class BankAccount
{
private:
    int balance = 5000;

public:
    void showBalance()
    {
        cout << "Balance = " << balance << endl;
    }
};

int main()
{
    BankAccount account;

    account.showBalance();

    return 0;
}
Output
Balance = 5000
Student Understanding
balance → Hidden (private)
showBalance() → Accessible (public)

Therefore data is protected.

2. Getters and Setters
Simple Explanation

Sometimes we want to access private data safely.

For that we use:

Getter

Gets (reads) the value.

Setter

Sets (updates) the value.

Real-World Example
Mobile Phone Password

You cannot directly modify the password stored inside the phone.

Instead:

Change Password option → Setter
View Profile option → Getter
Simple Program
#include <iostream>
using namespace std;

class Student
{
private:
    int age;

public:
    void setAge(int a)
    {
        age = a;
    }

    int getAge()
    {
        return age;
    }
};

int main()
{
    Student s;

    s.setAge(20);

    cout << "Age = " << s.getAge();

    return 0;
}
Output
Age = 20
Student Understanding
setAge() → Setter
getAge() → Getter

Private data can be accessed safely.

3. Abstraction
Simple Explanation

Abstraction means:

Show only important information and hide unnecessary details.

User knows WHAT to do but doesn't know HOW it works internally.

Real-World Example
Car

You know:

Start Car
Accelerate
Brake

You don't need to know:

Fuel Injection
Piston Movement
Combustion Process

Those details are hidden.

This is Abstraction.

Simple Program
#include <iostream>
using namespace std;

class Car
{
public:
    void start()
    {
        cout << "Car Started" << endl;
    }
};

int main()
{
    Car c;

    c.start();

    return 0;
}
Output
Car Started
Student Understanding
User sees:
start()

Hidden:
Engine operations
Fuel system
Battery operations

That's Abstraction.

4. Friend Function
Simple Explanation

Normally private members cannot be accessed outside the class.

But a friend function gets special permission to access private data.

Real-World Example

Imagine:

Principal Room

Normally students cannot enter.

But:

Office Staff

has special permission.

Friend Function works the same way.

Simple Program
#include <iostream>
using namespace std;

class Test
{
private:
    int num = 100;

public:
    friend void display(Test t);
};

void display(Test t)
{
    cout << "Number = " << t.num;
}

int main()
{
    Test obj;

    display(obj);

    return 0;
}
Output
Number = 100
Student Understanding
display()
↓
Not inside class
↓
Still accesses private data
↓
Because it is a friend
5. Friend Class
Simple Explanation

Instead of giving permission to one function,

we give permission to an entire class.

Real-World Example

A company allows:

HR Department

to access employee records.

Every HR employee gets access.

Similarly:

Class B

can access private members of

Class A
Simple Program
#include <iostream>
using namespace std;

class A
{
private:
    int salary = 50000;

    friend class B;
};

class B
{
public:
    void show(A obj)
    {
        cout << "Salary = " << obj.salary;
    }
};

int main()
{
    A a;
    B b;

    b.show(a);

    return 0;
}
Output
Salary = 50000
6. Basics of Pointers
Simple Explanation

A pointer stores the address of another variable.

Real-World Example

Imagine your house.

House = Variable
Address = Pointer

People find your house using the address.

Similarly:

Pointer stores address of variable
Diagram
num = 10

Memory:
2000 → 10

ptr = 2000

Pointer stores address.

Simple Program
#include <iostream>
using namespace std;

int main()
{
    int num = 10;

    int *ptr = &num;

    cout << "Value of num = " << num << endl;

    cout << "Address = " << ptr << endl;

    cout << "Value using pointer = " << *ptr << endl;

    return 0;
}
Output
Value of num = 10
Address = 0x61ff08
Value using pointer = 10
Student Understanding
& → Address Of Operator

*ptr → Value at Address
7. this Pointer
Simple Explanation

Every object automatically has a special pointer called:

this

It points to the current object.

Real-World Example

Imagine a teacher says:

"My classroom"
"My students"

"My" refers to the current teacher.

Similarly:

this

refers to the current object.

Simple Program
#include <iostream>
using namespace std;

class Student
{
private:
    int age;

public:
    void setAge(int age)
    {
        this->age = age;
    }

    void display()
    {
        cout << "Age = " << age;
    }
};

int main()
{
    Student s;

    s.setAge(20);

    s.display();

    return 0;
}
Output
Age = 20
Why use this?

Without this:

age = age;

Compiler gets confused.

Which age?

Parameter age?
or
Class variable age?

Using:

this->age = age;

means:

Current object's age = Parameter age
